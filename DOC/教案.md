# MediCare 智慧医疗门诊管理系统 — 教师指导教案

> **课程名称**：Vue 3 + Spring Boot 全栈开发与前后端分离实战
> **项目案例**：MediCare 智慧医疗门诊管理系统
> **课时**：12 课时（每课时 45 分钟）+ 课后分组开发时间
> **授课对象**：有 Java SE 基础、希望掌握 Web 全栈开发的学员
> **授课方式**：理论讲解 + 代码演示 + 学员实操（3-5 人小组协作）
> **版本**：v3.0（Vue + SpringBoot 前后端分离版）

---

## 一、教案核心设计

### 1.1 教学模式

本教案采用**"脚手架 + 分组协作"**的教学模式：

- **老师**：提供前后端脚手架代码（基础框架），演示一个完整模块的开发过程，指导学生分组实现其余模块
- **学生**：以 3-5 人小组为单位，在脚手架上独立完成分配模块的后端 API → 前端页面的开发
- **协作**：各小组通过 Git 分支协作，按模块依赖顺序集成联调

### 1.2 脚手架说明

老师向学生分发 `medicare-web/`（前端）和 `medicare-server/`（后端）两个目录，包含已封装好的基础框架代码。学生**禁止修改**脚手架中的核心配置文件。

**后端脚手架包含**：

| 类别 | 文件/目录 | 说明 | 是否可修改 |
|------|----------|------|-----------|
| 项目配置 | `pom.xml` | Maven 依赖、Spring Boot 插件 | ❌ |
| 入口类 | `MediCareApplication.java` | Spring Boot 启动类 | ❌ |
| 工具类 | `ConnectionConfig.java` | 数据源配置 | ❌ |
| 认证拦截 | `AuthInterceptor.java` | Session + Cookie 认证 | ❌ |
| 权限切面 | `RoleCheckAspect.java` | @RequireRole AOP 校验 | ❌ |
| 全局异常 | `GlobalExceptionHandler.java` | 统一异常处理 | ❌ |
| 实体模型 | `entity/*.java` (11个) | 所有 JPA 实体 | ❌ |
| 数据库脚本 | `sql/schema_baseline.sql` | 11 张表 DDL + 初始数据 | ❌ |
| 日志配置 | `logback.xml` | SLF4J/Logback | ❌ |

**前端脚手架包含**：

| 类别 | 文件/目录 | 说明 | 是否可修改 |
|------|----------|------|-----------|
| 项目配置 | `package.json`, `vite.config.ts` | Vite + Vue 3 工程 | ❌ |
| 入口文件 | `src/main.ts`, `src/App.vue` | Vue 应用入口 | ❌ |
| 路由 | `src/router/index.ts` | Vue Router 路由表 | ⚠️ 仅添加路由 |
| 请求封装 | `src/api/index.ts` | Axios 封装 + 拦截器 | ❌ |
| 类型定义 | `src/types/index.ts` | TypeScript 类型 | ⚠️ 可扩展 |
| 布局组件 | `src/views/layout/` | 主布局/侧边栏 | ⚠️ 仅添加导航 |
| 样式文件 | `src/assets/` | 全局样式 | ⚠️ 可扩展 |

**学生需要创建**：
- 后端：Repository 接口、Service 类、Controller 类、DTO
- 前端：API 模块、Vue 页面组件、弹窗组件

### 1.3 小组分工

| 小组 | 负责模块 | 依赖 | 技术要点 | 任务书 |
|------|----------|------|----------|--------|
| 老师演示 | 基础数据（科室/医生/排班） | 无 | 完整 CRUD + REST API 演示 | `DOC/tasks/task-teacher-基础数据演示.md` |
| A 组 | 患者管理 | 无 | el-Table + Dialog 复用 | `DOC/tasks/task-A-患者管理.md` |
| B 组 | 挂号预约 | A 组 + 老师 | 事务控制、弹窗联动 | `DOC/tasks/task-B-挂号预约.md` |
| C 组 | 医生工作站 + 病历 | B 组 + 老师 | 状态机、表单 | `DOC/tasks/task-C-医生工作站与病历.md` |
| D 组 | 药品库存 + 处方 | C 组 | 操作列、复杂事务 | `DOC/tasks/task-D-药品库存与处方.md` |

### 1.4 模块依赖与接口约定

```
基础数据（老师） ─┬─→ B 组（挂号预约）─┬─→ C 组（医生工作站+病历）─┬─→ D 组（处方管理）
                │                     │                           │
A 组（患者）─────┘                     └───────────────────────────┘
```

各小组开发前必须**约定 REST API 接口**（路径、方法、参数、响应），以便独立开发和后期集成。

## 二、Git 协作规范

### 2.1 仓库初始化（课时 1 完成）

```bash
# 老师创建中央仓库（GitHub / Gitee / GitLab）
git init --bare medicare.git

# 各小组克隆仓库
git clone <仓库地址>
```

### 2.2 分支策略

```
main （保护分支，仅通过 PR 合并）
  ├── feature/patient-mgmt      (A 组)
  ├── feature/registration      (B 组)
  ├── feature/doctor-workstation (C 组)
  └── feature/pharmacy          (D 组)
```

**规范**：
- `main` 分支受保护，禁止直接推送
- 每个模块一个 `feature/模块名` 分支
- 开发完成后发起 Pull Request，老师或其他组代码审查后合并

### 2.3 提交规范

```
feat: 新增患者管理模块后端 API
feat: 新增患者管理前端页面
fix: 修复挂号状态显示错误
refactor: 提取公共弹窗组件
docs: 更新接口文档
```

### 2.4 合并冲突处理

当两个小组同时修改了 `router/index.ts`（添加路由）时：
1. 先 `git pull origin main` 获取最新代码
2. 如有冲突，手动编辑解决
3. `git add . && git commit` 提交解决后的代码

## 第 7 课时：Axios 异步请求与前后端通信

### 一、教学目标

1. 理解前后端分离的 HTTP 通信机制
2. 掌握 Axios 封装与请求拦截器的标准模式
3. 能独立完成带加载反馈的数据请求

### 二、教学重点

- 为什么不能用同步请求阻塞 UI
- Axios 实例配置：`baseURL`、`timeout`、`withCredentials`
- **标准模板**：所有模块的数据请求都用这个模式

### 三、教学过程

#### 4.1 导入：痛点演示（5 min）

**老师活动**：故意在 UI 线程执行同步请求，演示界面卡死。

#### 4.2 讲解：Axios 标准模板（15 min）

**老师活动**：展示标准代码模板，强调"所有模块都用这个模板"：
```typescript
// api/patient.ts
import request from './index'

export function listPatients() {
  return request.get('/patients')
}

// Vue 组件中
async function loadData() {
  try {
    const res = await listPatients()
    tableData.value = res.data
  } catch {
    ElMessage.error('加载失败')
  }
}
```

#### 4.3 学生实操（25 min）

**学生活动**：
1. 创建一个按钮【加载数据】

### 5.3 常见学员问题预案

| 问题 | 解答要点 |
|------|----------|
| "为什么不给完整代码？" | 脚手架已经提供了 60% 的框架代码，你们只需要写 Repository/Service/Controller/Vue 页面，这正是需要学习的核心技能 |
| "其他组的模块没完成，我怎么联调？" | 开发前先约定 REST API 接口，对方未完成时用 mock 数据代替 |
| "JPA 查询报错 ConverterNotFoundException？" | 原生 SQL 查询返回 VO 时，必须使用接口投影（interface），不能用 class |
| "Git 合并冲突了怎么办？" | 保留双方修改，冲突标记 `<<<<<` 要全部删除干净 |
| "我的模块依赖 A 组，A 组没做完" | 先用固定的测试数据开发，等 A 组完成后替换为真实接口调用 |
| "前端报 401 未登录？" | 检查是否先调用登录接口，以及 Axios 是否携带 Cookie (`withCredentials: true`) |
| "后端改完代码前端没生效？" | 后端 Spring Boot 需要重启才能加载新代码；前端 Vite 热更新通常自动生效 |
